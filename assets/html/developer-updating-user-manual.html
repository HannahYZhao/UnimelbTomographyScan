How can I update the user manual?

To update this user manual, there are 3 steps.

  1. For each of the user manual page you want to add, create an HTML file for it under "assets/html" in the project file structure. (Note: If you are just updating some of the HTML files and are not adding new ones, you can skip the first two steps). Despite that QWidget may not support some of the advanced tags and styles that are normally used in web development, most of the very basic ones (e.g. unordered and ordered lists, bold, italic, etc.) work.
  2. After creating HTML file(s), they need to be declared in userManual.qhp. Alongside with the list of section tags, add a new section tag where title="[The title that will be shown in the content table on the left hand side", and ref="[the-file-name].html". The section tags are hierarchical, so you can create subpages by wrapping sections inside sections. You also need to include a file tag with the file name in it inside the files tag.
    In summary, the two lines that needs to be added are
    <section title="[title]", and ref="[file-name].html"></section>
    and <file>[file-name].html</file>.
    Save the qhp file.
  3. The third step is to compile the qhp and qhcp files. To compile the two files, you will need to find the qhelpgenerator tool on your local machine. If you already have QtPy installed in your Python site-packages, there should be a qt5_applications too, and the qhelpgenerator should be inside the Qt/bin folder in the qt5_applications. Take my machine (macOS) as an example, my qhelpgenerator is in /.../python3.9/site-packages/qt5_applications/Qt/bin.
    Copy the path of the qhelpgenerator, and run /path-to-qhelpgenerator/qhelpgenerator /path-to-qhcp/userManual.qhcp -o /path-to-qhc/userManual.qhc in your terminal. If it's compiled successfully, the new user manual page(s) should appear the next time you start the app.
