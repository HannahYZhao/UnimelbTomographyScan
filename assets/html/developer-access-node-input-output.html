How to I access the Inputs and Outputs for a Node?

While inside the update_event method, you can call self.input(i) to access the input for the (i+1)th port for that node. As usual in coding, the first port is the 0th one. This should be familiar as it is the same as indexing an array in most coding languages. For example, if you are getting a folder path on the 2nd port, you would call self.input(1)

In contrast, if you want to set the output of a node, you would need to call self.set_output_val(i, data) to send data to the (i+1)th port for that node.

The order of these ports are the same as when they are first defined in the init_inputs and init_outputs variables, as the order here defines the order they are displayed on the node.

There is one thing to note though when sending data between nodes: our system wants the data to be packaged within a dictionary that allows us to determine the type of the data being sent. More specifically, the dictionary contains a value "type" that says what type of data this dictionary contains, and a "data" value which contains the actual data being sent. This dictionary can be created using getNewTypedData as defined in procBuilder/pbUtil.py, and the data gotten from inside one by calling dataDictionary["data"]
